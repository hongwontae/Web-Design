** 핫스팟
=> 프로토타입에서 사용자가 클릭하거나 인터랙션을 발생시키는 부분입니다. (트리거 역할을 하는 객체)
** 데스티네이션
=> 사용자가 핫스팟을 동작시켰을 떄 이동하게 될 목적지입니다. (이동할 프레임이나 오버레이 화면)
=> 항상 데스티네이션은 Frame이여야 합니다.

1. Prototype
1-1. 실제 앱처럼 화면 전환을 연결하고 흐름을 보여주는 기능입니다.


2 Interaction
2-1. 사용자 행동에 따라 어떤 일이 벌어지는지 설정하는 부분입니다.
2-2. 트리거 -> 액션 -> Destination 3 단계 프로세스로 진행됩니다.
=> 트리거는 사용자 행동, 액션을 동작이나 애니메이션, Destination은 목적지를 의미합니다.
2-3. 인터랙션을 통해 모든 동작을 제어할 수 있습니다. 트리거, 액션, destination 전부..


3. 연결
3-1. 두 개의 프레임을 생성합니다.
3-2. 하나의 프레임에서 오른쪽 Prototype 패널을 클릭합니다.
3-3. Flow starting point를 클릭합니다.
3-4. 설정된 프레임에서 설정 안 된 프레임으로 선을 연결합니다.
3-5. 선을 클릭하면 interaction 설정 속성으로 제어할 수 있습니다.


4. Trigger
4-1. Click, Tap
=> 클릭하여 액션을 일으킵니다. (Click은 desktop, Tap은 phone)

4-2. while hovering
=> 핫스팟에 마우스를 갖대되면 액션을 일으킵니다.

4-3. while pressing
=> 4-1과 동일

4-4. Key/Gamepad
=> 특정 키를 눌러서 액션을 일으킵니다.

4-5. After Delay
=> 특정 시간 이후에 액션을 일으킵니다.


5. Action
5-1. Navigate to
=> Destination으로 단순히 이동합니다.

5-2. Change to
=> 컴포넌트 + variant 요소가 존재할 떄 사용할 수 있습니다.
=> 프레임 내에 컴포넌트 요소를 가져오고 버튼을 클릭하고 Interaction을 클릭합니다.
=> trigger, chage to를 선택하고 변경할 변형을 선택합니다.

5-3. Overlay
=> 기존 프레임을 전체를 바꾸는 것이 아니라 기존 프레임에 덧대는 표현방법입니다.
    1. Open Overlay
    => 새로운 오버레이를 원하는 프레임에 띄우는 동작입니다.
    2. swap Overlay
    => 1의 오버레이를 교체하는 동작입니다. (위치를 동일하게)
    3. Overlay Option
        3-1. Close when clicking outside
        => 오버레이 외부를 클릭하면 오버레이가 사라집니다.
        3-2. Background
        => 오버레이를 제외한 화면을 불투명색으로 바꿉니다.
    4. Scroll to
    => trigger가 발생되면 원하는 원하는 공간(프레임)으로 이동하는 기능입니다.
    5. Open Link
    => trigger가 발생되면 링크를 따라 이동하는 기능입니다.
    6. back
