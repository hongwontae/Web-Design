1. WebFlow의 block 태그들은 실제 높이는 0이지만 편리를 위해 일정 높이를 가시적으로 보여줍니다.

2. 피그마는 px 단위를 사용해서 고정된 디자인을 생성합니다.
3. WebFlow에서 px 단위를 사용하면 캔버스에 콘텐츠가 넘치거나 작아집니다.
=> 반응형으로 만들어야 합니다.

4. CSS 단위
4-1. px
=> 모든 화면에 고정된 크기를 가지는 단위입니다.
=> 브라우저 너비가 500px일 떄 요소의 너바가 1000px이라면 브라우저 스크롤을 이용해야 요소를 볼 수 있습니다.


4-2. auto
1. width : auto 
=> 가능한 부모의 너비를 전부 차지합니다.
=> 명확한 값이 부모나 자신에게 없고 block 요소라면 너비 전체를 차지합니다. (block 요소가 아니라면 텍스트만큼)
=> 명확한 값이 부모에게 있다면 부모의 너비 크기를 전부 차지합니다.

2. height : auto
=> height auto는 무조건 자신의 콘텐츠 길이만큼만 차지합니다.


4-3. %
1. width : n%
=> 부모의 너비를 %만큼 차지합니다.
=> 이 떄 부모의 크기를 정확히 명시하지 않아도 됩니다. (부모의 width : auto이고 자식 width이 50%라면 브라우저 50% 차지)

2. height : n%
=> 부모의 높이를 %만큼 차지합니다.
=> 이 떄 부모의 높이를 정확히 명시해야 적용됩니다. (부모의 height : auto이고 자식 height이 n%라면 0이나 무효가 됩니다.)


4-4. min-max
1. min-width : 요소의 최소 너비, 이 값보다 작아질 수 없습니다.
2. max-width : 요소의 최대 너비, 이 값보다 커질 수 없습니다.
=> 자식 : 80%, max-width : 600px, min-width : 300px 일 떄
case1. 부모가 1000px이라면 자식이 800px이며 최대값 600px을 넘지 않아야 함으로 자식은 600px을 갖습니다.
case2. 부모가 400px이라면 자식이 320px이며 어느 조건도 위배하지 않기에 320
case3. 부모가 300px이라면 자식이 240px이며 최소 300px를 넘어야 하기에 300px을 갖습니다.


3. min-height : 요소의 최소 높이, 이 값보다 작아질 수 없습니다.
4. max-height : 요소의 최대 높이, 이 값보다 커질 수 없습니다.

5. 만약 width과 height 없이 그냥 max, min 값만 추가했을 떄
=> default은 auto라서 이 값을 기준으로 판단하면 됩니다.


4-5. vw, vh
1. vw 
=> 화면 너비
2. vh
=> 화면 높이
3. height이 auto이고 min-height이 100vh라면?
=> height이 콘텐츠만큼 차지하나 100vh을 넘지 않으면 강제로 100vh를 가진다.
